# process.html 演示页设计规格

日期：2026-07-03

状态：历史设计规格，记录 `process.html` 第一版的设计背景和验收目标，不代表 2026-07-14 的项目开发进度。当前实现以根目录 `process.html` 和 `assets/process/` 为准，当前工程状态见 `docs/project-state.md`。

当前实现相对本规格的扩展：

- 在“小组分工”和“当前进度”之间增加“我组优势”区块。
- 当前 HTML 实际引用 8 张本地图片：小组主视觉、校园线稿、配额截图、GitHub/PRD/Issue 截图和两张透明笔触；`brand-smear.png`、`hero-red-brush.png` 是未直接引用的非透明备份。
- 页面仍是无框架、无网络资源依赖的单页自然滚动实现；“当前进度”内容冻结为 2026-07-03 汇报语境，不应作为项目现状来源。

## 背景

本设计面向 `EcoCampus 校园闲置物品智慧流转小程序` 的课堂项目进度汇报。老师要求各组在 2026-07-03 下午 3 点于 201 室介绍项目情况，内容包括组队、内部分工、项目开发进度和后续计划，并需要准备 PPT。

本项目先使用根目录 `process.html` 承担演示文稿用途。第一版不是正式应用页面，也不接入业务接口，而是一页自然滚动的演示页面，用于现场投屏和课后查看。

## 目标

- 用单个 HTML 文件呈现项目汇报内容。
- 保留产品发布会式的视觉冲击，不做传统 PPT 点击翻页。
- 明确展示项目定位、团队分工、当前进度和后续计划。
- 通过 Falcons VS Va Bro 手绘图形成小组识别。
- 页面在桌面投屏和移动端 H5 浏览时都保持可读。

## 非目标

- 不实现真正的业务功能。
- 不接入后端接口。
- 不引入前端框架、构建工具或 npm 依赖。
- 不制作完整 PPT 文件。
- 不公开展示成员邮箱或 GitHub 账号。
- 不做自动轮播或点击跳转式幻灯片。

## 页面形式

页面采用一页纵向自然滚动结构。内容按发布页节奏组织，但每个模块信息量控制在 3 分钟汇报可讲完的范围内。

布局不使用整屏吸附滚动，也不使用自动轮播。用户可通过鼠标滚轮、触控板或移动端滑动自然浏览。

## 视觉方向

整体采用产品发布会风格，而不是传统课堂 PPT 模板。设计应有年轻的小组识别，但正文信息必须稳、清晰、可投屏。

### 色彩

- 纸白：`#f7f8f5`，用于页面主背景。
- 墨黑：`#171717`，用于正文和主要标题。
- Falcons 绿：`#0b8f22`，来自手绘图左侧绿色形象，用于项目强调、进度标记和局部边线。
- Va Bro 红：`#e4231f`，来自手绘图右侧红色形象，用于关键视觉对比和小面积强调。
- 淡蓝灰：`#e9eef5`，用于材料卡片和截图背景。
- 中性灰：`#767676`，用于辅助说明和元信息。

### 字体

使用系统字体，不依赖外部字体资源：

- 中文与正文：`system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- 标题加粗，字号拉开层级。
- 数据、链接、标签可使用同一系统字体的小号字重，不单独引入等宽字体。

### 标志性元素

手绘 Falcons VS Va Bro 图是第一版的核心识别元素。

处理方式：

- 首屏大面积展示，作为主视觉。
- 中间内容模块可复用局部色彩和线条感，但不让背景图压住文字。
- 结尾再次出现，完成视觉闭环。

## 资产策略

需要将用户提供的图片复制到项目内，避免页面引用临时目录或下载目录。

建议路径：

```text
assets/process/
├── falcons-vs-va-bro.png
├── prd-snapshot.png
└── issue-report-snapshot.png
```

资产用途：

- `falcons-vs-va-bro.png`：首屏和结尾主视觉。
- `prd-snapshot.png`：当前进度中的产品文档材料。
- `issue-report-snapshot.png`：当前进度中的测试表材料。

图片加载失败时，页面应显示有意义的文字替代说明。

## 实现边界

第一版只新增：

- `process.html`
- `assets/process/` 下的三张图片资产

`process.html` 使用内联 CSS 和必要的少量内联 JavaScript。JavaScript 仅用于轻量交互增强，例如返回顶部、滚动进度条或进入页面动画；如果没有必要，可以不写 JavaScript。

页面不依赖网络资源。外链只用于跳转 GitHub，不用于加载页面必要资源。

## 响应式要求

桌面端：

- 适合 16:9 投屏。
- 首屏标题和主视觉醒目。
- 分工和进度模块可在一屏内快速扫读。

移动端：

- 页面不出现横向滚动。
- 角色分工从多列布局降为单列。
- 图片缩略图按容器宽度自适应。
- 链接按钮可点击区域足够大。

## 可访问性与容错

- 所有图片需要 `alt` 文本。
- 链接需要清晰可见的焦点状态。
- 文本和背景对比度足够用于投屏。
- 图片加载失败时保留文字信息。
- 页面不应依赖动画传达关键信息。

## 验收标准

- 根目录存在 `process.html`，可直接用浏览器打开。
- 页面为自然纵向滚动，不需要点击翻页。
- 首屏强展示 Falcons VS Va Bro 手绘图。
- 页面包含项目背景、团队分工、当前进度、技术路线和后续计划。
- 团队分工不展示邮箱或 GitHub 账号。
- 技术路线只做简洁说明，不展开复杂架构。
- 后续计划简短，不写详细排期。
- 桌面宽屏和手机宽度下均无明显重叠、溢出或不可读文本。
