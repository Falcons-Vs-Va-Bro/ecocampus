# 线上功能演示录制总分镜

状态：已落地到 `recording/record-online-message.mjs`、`recording/record-online-flows.mjs` 和 `recording/lib/recording-kit.mjs`，仅使用线上真实环境。

## 总体叙事

建议最终按以下顺序剪辑，形成“身份可信 → 供给产生 → 交易完成 → 平台治理 → 主动发现”的完整故事：

1. 校园双重认证与常用地址；
2. 商品图文发布与管理员审核；
3. 移动检索、收藏、预约与订单状态流转；
4. 桌面/移动双账号私信同步；
5. 后台违规下架、黑名单、看板与类目入口；
6. 发布求购并由系统匹配在售商品。

旁白统一用 `say -o` 生成离线 AIFF，再按分镜时间混入 MP4。每段同时输出 SRT 和 JSON，便于后续人工改词或替换 TTS。

## 线上数据安全边界

- 所有 mutation 使用 seed 或专用录制账号，标题明确带录制标记。
- access token 只保留在内存；成片元数据只写掩码账号和业务 ID。
- 地址和求购在录制后清理或关闭；黑名单状态在正常和异常路径均恢复。
- 商品发布、成交、违规下架、认证账号和私信是业务本身不可删除的记录，会作为明确标记的演示审计轨迹保留。
- 任一关键状态未从真实 API 读到时，脚本停止合成，不能用前端视觉假装成功。
